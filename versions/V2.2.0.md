# ZNAMBO Telegram Assistant V2.2.0

Status: released

## Scope

- OpenAI became mandatory for non-command natural-language turns.
- Added strict Responses API tool calling and fail-closed behavior.
- Added per-turn AI telemetry in `assistant.audit_log`.
- Added `/aihealth`, expanded `/debuglast` and safe health telemetry.
- Connected model proposals to ActionPlan creation and existing-item policy updates.
- Added management buttons and follow-up catch-up delivery.

## Production proof

- Real OpenAI health call returned model, response ID, tokens and valid tool output.
- Exact daily list produced `event`, `event`, `training` at 10:00, 13:00 and 22:00.
- Exact relative reminder instruction updated the same three item IDs.
- Model-driven catch-up follow-up was delivered automatically through cron-job.org.
- `fallbackUsed` remained false in successful and failed-closed traces.

## Validation

- 20 test files and 64 tests passed.
- Lint passed.
- Production build passed.

## Known limitations

- Item updates represented reminder configuration but not explicit completion or schedule changes.
- Hybrid show-plus-create requests could copy existing context into a new ActionPlan.
