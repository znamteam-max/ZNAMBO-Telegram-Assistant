# V2.10.0 - Recurring Policy Execution and End-of-Day Fix

Release date: 2026-06-14

## Implemented

- Added canonical weekly recurrence rules such as `weekly:MO@10:00`.
- Added one logical monthly day-range policy such as
  `monthly_days:15,16,17,18,19@12:00`.
- Recurring requests without a time now produce a typed draft and targeted time clarification.
- Multi-reminder requests preserve separate titles and policies instead of collapsing into one
  generic task.
- Cadence-only phrases such as `Каждый понедельник` are rejected as task titles.
- Reminder setup accepts stop-condition-only replies such as `Пока не выполню`, stores them in the
  active draft, and asks only for the next missing field.
- The recurring scheduler calculates the next weekly or monthly occurrence in the user's timezone.
- `сегодня до конца дня` and `завтра до конца дня` resolve to local 23:59 without a UTC date shift.
- `/debuglast` reports the failed recurring-policy tool, safe reason, and missing field.
- `/admin_repair_v2100 preview|apply` archives only the known cadence-title garbage task and its
  generated policies. It never deletes Yandex Calendar objects.

## Database

No migration is required. Canonical recurrence rules use the existing
`assistant.reminder_policies.recurrence_rule` field. Typed draft state uses the existing
`assistant.agent_actions` table.
