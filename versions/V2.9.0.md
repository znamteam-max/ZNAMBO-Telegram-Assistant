# V2.9.0 - Deadline Semantics and Due Tasks

Release date: 2026-06-14

## Implemented

- Natural-language deadline markers create tasks with `dueAt` instead of accidental scheduled
  blocks. The mandatory OpenAI call still runs first; a post-AI semantic normalizer enforces the
  separate deadline model.
- `завтра до 14:00`, weekday deadlines, explicit dates, and mixed work-block plus deadline input
  are supported.
- Plan rows and task cards distinguish scheduled work time from a deadline and never classify a
  deadline-only task as currently running.
- Known project names are normalized conservatively, including `Больше`, `Централ Парк`, and
  `ЧМ-26`.
- Newly created deadline tasks offer reminder presets. Future deadlines offer morning, two-hour,
  and 30-minute options; same-day deadlines offer soon, one-hour, and 30-minute options.
- Item-card edits support setting, changing, clearing, and combining a deadline with a scheduled
  work interval.
- `/admin_repair_v290 preview|apply` repairs the exact June 14 production misparse without deleting
  or changing Yandex Calendar objects.

## Database

No migration is required. V2.9.0 uses the existing separate `planner_items.start_at`,
`planner_items.end_at`, and `planner_items.due_at` fields.
