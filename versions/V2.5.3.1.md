# V2.5.3.1 - Normal CalDAV Sync Resilience

Date: 2026-06-12

V2.5.3.1 is a focused hotfix for normal Yandex CalDAV event synchronization. It does not
replace the planner, OpenAI ActionPlan, reminder engine, reconciler, or runner lock.

## Implemented

- Normal event sync and `/calendar_test` now use the same hardened CalDAV request lifecycle.
- Normal events use one deterministic object URL and a matching ICS UID.
- Added separate request, read-back, and total sync timeout configuration.
- Slow immediate sync is aborted, the planner item remains saved, and a retry job is queued.
- Retry first performs GET on the same object URL; PUT is used only after confirmed not-found.
- Added automatic calendar retry processing to the existing minute runner.
- Added `/calendar_retry_failed`, item-card retry/debug/disable controls, and V2.5.3 calendar repair.
- Extended safe `/calendardebug` normal-sync and queue metrics.
- Added correct Russian pluralization for unresolved dashboard items.
- Applied `drizzle/0006_calendar_sync_resilience.sql` to production Neon.
