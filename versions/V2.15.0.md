# V2.15.0 - Release Notification and Deploy Completion

Production URL: https://znambo-telegram-assistant.vercel.app

## Implemented

- Added `assistant.release_notifications` with unique
  `(version, commit_sha, environment)` idempotency.
- Release notification is blocked until production health, exact version and commit, Telegram
  webhook, runner, migration evidence, smoke evidence, and handoff completion are confirmed.
- Duplicate calls return `already_sent` without another Telegram message.
- Failed Telegram sends remain retryable.
- Same-version hotfixes require explicit `allowHotfix`.
- Added owner commands `/version`, `/release`, `/release_notes`, `/changelog`, and
  `/release_notify`.
- Added protected admin action `release_notify` under `/api/admin/repair`.
- Added safe structured release notes and secret redaction.
- `/api/health` exposes safe latest notification status only.
- Added migration `drizzle/0009_release_notifications.sql`.
