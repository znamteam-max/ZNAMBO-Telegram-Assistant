# V2.5.4.1 - Item Card Edit Sessions

Date: 2026-06-12

V2.5.4.1 fixes the production bug where a reply to an item card edit was intercepted by the
global numbered-list reschedule guard.

## Implemented

- `manage:edit` and `manage:reschedule` now create an active item edit session.
- Natural-language replies inside an active edit session are applied to the current item and do not
  require a list number.
- Compound item edits support title rename, date/time move, kind inference for event-like titles,
  and `nag_until_ack` reminder policy setup.
- The exact orthodontist command is parsed as one preview:
  title rename, Monday 08:00 schedule, hourly reminders from 08:00 until done.
- Russian date parsing now handles Monday/Tuesday forms, explicit `15.06`, `10.20`, and past
  same-day confirmation.
- Slash commands clear active edit sessions so `/dashboard` and `/plan` cannot be hijacked by a
  stale item card context.
- Dashboard no longer shows active same-day past items as upcoming Today items; they move to
  unresolved.
- Item edit actions are undoable with item snapshots and reminder policy restoration/cancellation.
- Calendar sync remains best-effort after edit mutations and does not block saving.

## Remaining Limitations

- The item edit parser is intentionally scoped to active item-card sessions; broad planning still
  goes through Jarvis/OpenAI.
- Complex edits beyond title/time/hourly-until-done reminders may still need a follow-up message.
