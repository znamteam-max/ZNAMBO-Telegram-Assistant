# ZNAMBO Telegram Assistant V2.5.2

Release date: 2026-06-11.

## Scope

V2.5.2 makes visible planner entities editable, replaces raw priority codes with human importance,
repairs Russian weekday appointments deterministically, prevents silent duplicate no-ops, and
guards future campaign events from premature completion.

## Changes

- Added universal entity references and cards for planner items, reminder policies, campaigns,
  campaign items, history items, and legacy orphans.
- Added opening buttons to dashboard, task views, reminder center, long-term views, and history.
- Added human importance labels, `importanceMode`, separate base/effective priority, and urgency
  explanations without overwriting base importance.
- Added semantic `distant_priority` and `campaign_active` timeline classes.
- Added deterministic Russian weekday and appointment-clock parsing. On 2026-06-11,
  `во вторник к 10.20` resolves to 2026-06-16 10:20.
- Medical/family appointments receive health metadata, automatic base importance 4, and reminder
  suggestions.
- Duplicate natural-language requests now reply and open existing records instead of silently
  doing nothing.
- Future campaign `Done` actions now require clarification; preparation completion does not
  complete the event or activate the next stage.
- Added `/admin_repair_v252 preview|apply` and protected HTTP equivalents. Repair fixes the
  orthodontist date, reuses V2.5.1 malformed/Central Park repair, marks legacy Drik orphans,
  archives old overdue records, marks old bot cards stale, and saves an audit rollback snapshot.

## Preserved Architecture

- One external minute scheduler.
- Distributed runner lease.
- Reconciler before claim.
- Anchor-grid-v2 interval schedules.
- Atomic action-plan item/policy/reminder commit.
