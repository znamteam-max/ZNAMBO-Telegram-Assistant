# ZNAMBO Telegram Assistant V2.5.1

Release date: 2026-06-11.

## Scope

V2.5.1 adds compact Telegram control UX, semantic timeline routing, editable priority and reminder
policies, correct upcoming-night parsing, explicit reminder-time guarantees, sequential Central Park
campaign behavior, and durable daily history without replacing the V2 scheduler architecture.

## Changes

- Added `syncCompactChat`: dashboard is sent before one current active reminder card; old bot cards
  are deleted or disabled through the bot-only message registry.
- Removed persistent success confirmations from normal Jarvis mutations and reminder callbacks.
- Removed persistent voice transcription status/full-transcript messages; added `/lasttranscript`.
- Added semantic `now/today/active_nag/soon/distant/long_term/campaign_waiting` classification.
- Added effective priority escalation and priority-sorted distant rendering.
- Added item and reminder-policy priority editing, interval editing, pause/resume/cancel, and policy
  version audit with `/undo` fallback.
- Added `/reminders` scopes, `/distant`, `/priority`, and interactive policy cards.
- Added deterministic NBA/upcoming-night repair for 03:30 plus explicit 01:30/02:00 reminders.
- Added fail-closed validation when reminder intent or explicit reminder times lack concrete reminders.
- Changed Central Park normalization so the second event does not begin reminders before the first
  event ends and grouped campaign policies in compact views.
- Added durable per-day snapshots in `assistant.audit_log` and `/history`, `/yesterday`, `/weeklog`,
  `/review`.
- Added owner-only `/admin_repair_v251 preview|apply`.

## Preserved Architecture

- One external minute scheduler.
- Distributed runner lease.
- Reconciler before claim.
- Anchor-grid-v2 interval schedules.
- Snooze as a separate one-off reminder.
- Atomic action-plan item/policy/reminder commit.
