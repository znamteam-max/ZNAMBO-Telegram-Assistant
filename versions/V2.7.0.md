# V2.7.0 - Calendar Import Hygiene and Reminder Capture Fix

Release date: 2026-06-13

## Implemented

- Clear natural-language reminder requests are normalized after the mandatory OpenAI proposal:
  exact-time reminders, relative one-time reminders, and open-ended hourly `nag_until_ack`.
- The anti-garbage validator accepts open-ended nag policies and returns a precise missing-field
  explanation instead of the generic ambiguity response.
- External Yandex events are classified on import. Service/test events and ended past events are
  hidden from the default Plan, while real future events remain visible.
- Plan buckets and conflict detection no longer place past/service events under `Скоро` or show
  conflicts for hidden/service/ended entries.
- `/calendar_cleanup preview|apply`, `/calendar_view`, and `/admin_repair_v270 preview|apply` provide
  safe local cleanup and visibility controls without deleting real Yandex Calendar data.
- Plan and reminder views reconcile active reminder policies before rendering.
- Voice transcription and planner-guard status are recorded through safe audit diagnostics and
  exposed in `/api/health` without message text or secrets.
- Calendar reads remain best-effort so an import failure cannot block the core Plan.

## Database

No migration is required. V2.7.0 uses existing JSON metadata fields in
`assistant.external_calendar_events` and `assistant.calendar_import_state`.
