# V2.5.4 - Unified Plan UX

Date: 2026-06-12

V2.5.4 unifies the Telegram planning interface and closes the destructive numbered-operation
incident without changing CalDAV, the reminder runner, its lease, or mandatory OpenAI planning.

## Implemented

- `/plan` and `/dashboard` show one canonical plan with today, tomorrow, the next seven days,
  conflicts, important work, long-term work, unresolved items, and reminder rules.
- `/tasks` uses the same canonical timeline and keeps numeric item-card controls plus management
  actions.
- `/reminders` is explicitly a reminder-rule view and links back to Plan and Tasks when empty.
- Item cards show human item types, reminders, importance, and `provider · calendar · status`.
- Every committed batch receives one optional post-create triage card.
- Event-like overlap detection uses `A.start < B.end && B.start < A.end`.
- Conflicts are visible after create/update and in the main Plan without blocking creation.
- Numbered deletion now creates a current-view preview and requires an explicit callback
  confirmation.
- Comma lists remain explicit lists, ranges require `5-8`, and clock ranges such as
  `10.00-11.00` are never interpreted as item indices.
- Mixed delete plus time update is previewed as one exact operation before any mutation.
- Confirmed deletion refreshes Tasks and Plan and exposes Undo.
- Deleting an item with a repeat policy asks whether to remove only the rule or the task and rule.
- Added conservative owner-only `/admin_repair_v254 preview|apply`.

## Remaining Limitations

- Text-number operations intentionally use only the latest saved Plan/Tasks view and expire with
  that view.
- Calendar synchronization remains best-effort and non-blocking.
- Archive/review buttons guide the user to the existing history/review flows; there is no separate
  web admin UI.
