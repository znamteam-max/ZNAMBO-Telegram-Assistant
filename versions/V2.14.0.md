## Implemented

- Incomplete weekly recurring reminders without a time now become typed clarification drafts.
  They do not create planner items, recurring policies, reminders, or action plans until the time
  is selected.
- Similar recurring policies are detected before saving. The bot asks whether to update the
  existing policy, create a separate policy, or cancel.
- Reminder setup labels now map to user intent: concrete time, before event, repeat, nag until
  done, multiple reminders, and advanced.
- Selected events can receive several before-event reminders from one message, for example
  `za den v 9 utra, za 2 chasa i za 30 minut`.
- Before-event reminder rendering now shows concrete offsets such as `za den v 09:00`,
  `za 2 chasa`, and `za 30 minut`.
- `/completed` and `/done` show completed items with pagination plus restore and archive actions.
- `/cleanup`, `/cleanup_chat`, and the cleanup keyboard show a preview before any Telegram UI
  cleanup. They never delete planner data or Yandex Calendar objects.
- Active past tasks are classified as overdue. Unresolved is reserved for broken, imported,
  orphaned, or explicit review data.
- Missing recurring-time audit/debug details now include failure reason, missing field, and a safe
  suggested prompt.
- Added `/admin_repair_v2140 preview|apply` for safe production cleanup. It changes zero Yandex
  Calendar objects.
- Added protected admin API actions `v2140_repair_preview` and `v2140_repair_apply`.
- No database migration was required.
