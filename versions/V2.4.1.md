# V2.4.1 Reminder Reliability Repair

Released to production on 2026-06-08.

## Scope

- Added policy reconciliation before every reminder claim.
- Kept one external minute scheduler; no per-task cron jobs were added.
- Fixed interval progression to use `occurrence.scheduledFor`, preserving the 30-minute grid.
- Added one-immediate catch-up without replaying every missed slot.
- Added inclusive interval-window ends and quiet-hours defaults/overrides.
- Added scheduler runtime health, `/cronhealth`, `/policydebug` and safe health fields.
- Added nested reminder setup and expanded reminder-card snooze controls.
- Isolated Telegram message cleanup from planner-item mutation.
- Added owner-only preview/apply repair for circle, Drik, mirror and ЖКХ legacy records.
- Added legacy diagnostics to `/reminders` and `/longterm`.
