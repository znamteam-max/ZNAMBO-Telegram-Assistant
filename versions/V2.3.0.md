# ZNAMBO Telegram Assistant V2.3.0

Status: implementation in progress

## Target behavior

- Complete only the item associated with the latest delivered follow-up.
- Represent item mutations as typed `configure`, `complete` and `reschedule` operations.
- Change start/end ranges without duplicate item summaries.
- Preserve active reminder policy when an event is rescheduled.
- Support `show plan + add one item` as a mutation followed by a view.
- Skip existing planner entries before committing a model-proposed ActionPlan.
- Show per-item management buttons on configured follow-up reminders.

## Regression scenarios

```text
Отлично! Выполнено, поставь это сделанным
Дай план на сегодня, добавь вечером подготовку к ЧМ дома
Поменяй подготовку к ЧМ на 22:00, а тренировку на 23:00
Эфир ВС с 13 до 20 сделай
```
